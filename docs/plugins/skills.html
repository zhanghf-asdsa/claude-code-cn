技能（Skills）

技能（Skills）是插件系统中用于添加特定领域知识和能力的组件。与命令和代理不同，技能会根据对话上下文自动激活。

什么是技能？

技能是一种上下文感知的扩展方式，具有以下特点：

  • 自动激活 - 根据对话内容智能启用
  • 领域专业 - 提供特定领域的专业知识
  • 上下文保留 - 可访问完整的对话历史
  • 组合使用 - 多个技能可以同时激活

技能与代理的区别

特性 技能（Skills） 代理（Agents）
激活方式 自动（上下文触发） 手动（用户指定）
上下文访问 完整对话历史 无（独立上下文）
工具权限 所有可用工具 受限工具集
适用场景 领域知识增强 任务执行

创建技能

在插件的 skills/ 目录中为每个技能创建子目录：

目录结构

skills/
├── code-review/
│   └── SKILL.md
├── react-dev/
│   ├── SKILL.md
│   ├── references/
│   │   ├── hooks.md
│   │   └── patterns.md
│   └── examples/
│       └── component.md
└── api-design/
    ├── SKILL.md
    └── best-practices.md

SKILL.md 格式

每个技能必须有 SKILL.md 文件：

---
name: code-review
description: 代码审查和最佳实践指导
version: 1.0.0
---

你是一位代码审查专家，专注于识别代码问题并提供改进建议。

## 核心原则

1. **安全性优先** - 检查常见安全漏洞
2. **性能意识** - 识别性能瓶颈
3. **可维护性** - 确保代码清晰易懂

## 审查流程

当用户请求代码审查时：

1. 读取变更的文件
2. 按严重程度分类问题
3. 提供具体的修复建议
4. 指出做得好的地方

## 问题分类

- **关键**：安全漏洞、数据丢失风险
- **重要**：性能问题、逻辑错误
- **建议**：代码风格、命名约定

## 输出格式

使用结构化格式：
```markdown
## 🔴 关键问题
[问题描述]

## 🟡 重要问题
[问题描述]

## 💡 建议
[改进建议]

## ✅ 优点
[做得好的地方]
```

技能元数据

name

技能的唯一标识符：

---
name: react-expert
---

description

技能的用途说明：

---
description: React 开发专家，提供组件设计、状态管理和性能优化建议
---

version

技能版本号：

---
version: 2.1.0
---

技能类型

领域专家技能

---
name: typescript-pro
description: TypeScript 高级用法和类型系统专家
---

专注于 TypeScript 的高级特性：
- 泛型约束
- 条件类型
- 类型推断
- 装饰器使用
- 类型体操技巧

工作流技能

---
name: tdd-workflow
description: 测试驱动开发工作流指导
---

TDD 开发流程：

1. **红** - 编写失败的测试
2. **绿** - 编写最小代码通过测试
3. **重构** - 改进代码质量
4. **重复** - 继续下一个功能

标准检查技能

---
name: accessibility-audit
description: 无障碍访问性检查和改进
---

WCAG 2.1 AA 标准检查：

- 键盘导航
- 屏幕阅读器支持
- 颜色对比度
- ARIA 属性使用
- 焦点管理

引用和示例

技能目录可以包含额外的参考资料：

references/

存储参考文档和最佳实践：

skills/react-pro/
├── SKILL.md
└── references/
    ├── hooks-patterns.md      # Hooks 使用模式
    ├── performance-tips.md     # 性能优化技巧
    └── common-pitfalls.md      # 常见陷阱

examples/

存储代码示例和用例：

skills/react-pro/
├── SKILL.md
└── examples/
    ├── custom-hook.md         # 自定义 Hook 示例
    ├── hoc-pattern.md         # HOC 模式示例
    └── compound-components.md # 组合组件示例

技能激活

技能根据以下条件自动激活：

  • 关键词匹配 - 用户请求中的术语
  • 文件上下文 - 正在操作的文件类型
  • 项目特征 - 项目配置和依赖
  • 对话历史 - 之前的交互内容

最佳实践

  1. 明确范围 - 每个技能专注于一个领域
  2. 详细指导 - 提供清晰的步骤和说明
  3. 示例丰富 - 包含实际代码示例
  4. 保持更新 - 随技术演进更新技能
  5. 测试验证 - 确保技能按预期工作
