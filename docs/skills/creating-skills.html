创建技能

技能（Skills）是可重用的提示词模板，可以让 Claude 专注于特定任务，提高效率和一致性。

什么是技能？

技能是一个 Markdown 文件，包含：

  • 元数据 - 技能名称、描述、触发条件
  • 提示词 - 给 Claude 的具体指令
  • 示例 - 可选的使用示例

技能文件结构

技能文件存放在 .claude/skills 目录：

.claude/
└── skills/
    ├── code-review.md
    ├── refactor.md
    └── test-helper.md

创建您的第一个技能

步骤 1：创建技能文件

在 .claude/skills 目录创建 code-review.md：

---
name: code-review
description: 审查代码质量、安全性和最佳实践
---

# 代码审查技能

请按照以下标准审查代码：

1. **代码质量**
   - 代码是否清晰易读？
   - 是否遵循了项目的编码规范？
   - 变量和函数命名是否恰当？

2. **安全性**
   - 是否存在常见的安全漏洞？
   - 用户输入是否正确验证？
   - 敏感数据是否得到保护？

3. **性能**
   - 是否存在性能瓶颈？
   - 算法复杂度是否合理？
   - 是否有优化空间？

4. **最佳实践**
   - 是否遵循了语言和框架的最佳实践？
   - 错误处理是否充分？
   - 测试覆盖是否足够？

请提供具体的改进建议，并解释原因。

步骤 2：使用技能

在会话中，您可以：

# 使用技能
请使用 code-review 技能审查以下文件

# 或者直接调用
/review src/app.js

技能元数据

技能文件的 YAML 前言可以包含以下字段：

字段 说明 必需
name 技能名称 是
description 技能描述 是
version 版本号 否
author 作者 否
tags 标签数组 否
trigger 触发条件 否

技能模板

使用模板可以快速创建技能：

---
name: {{技能名称}}
description: {{技能描述}}
tags: [{{标签1}}, {{标签2}}]
---

# {{技能名称}}

## 目标
{{描述这个技能的目标}}

## 使用场景
{{描述何时使用这个技能}}

## 步骤
1. {{步骤 1}}
2. {{步骤 2}}
3. {{步骤 3}}

## 注意事项
{{使用时需要注意的事项}}

技能最佳实践

  1. 明确目标 - 每个技能应该专注于一个特定任务
  2. 详细说明 - 提供清晰的步骤和期望
  3. 包含示例 - 添加输入输出示例
  4. 版本控制 - 将技能文件纳入 Git 管理
  5. 持续改进 - 根据使用反馈优化技能

共享技能

您可以将技能发布到：

  • GitHub - 创建技能仓库分享
  • npm - 打包为 npm 包
  • 团队 - 在团队内部共享
