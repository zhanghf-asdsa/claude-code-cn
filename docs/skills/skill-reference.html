技能参考

完整的技能开发参考，包括高级功能和最佳实践。

技能类型

1. 审查型技能

用于审查和评估代码、文档等。

---
name: security-review
description: 安全审查技能
tags: [security, review]
---

# 安全审查

请审查以下内容的安全问题：
- SQL 注入
- XSS 漏洞
- CSRF 保护
- 敏感数据泄露

2. 生成型技能

用于生成代码、文档、测试等。

---
name: test-generator
description: 自动生成单元测试
tags: [testing, generation]
---

# 测试生成器

根据给定的函数生成完整的单元测试：
1. 正常情况测试
2. 边界条件测试
3. 错误处理测试

3. 转换型技能

用于代码重构、格式转换等。

---
name: typescript-converter
description: JavaScript 转 TypeScript
tags: [typescript, conversion]
---

# TypeScript 转换器

将 JavaScript 代码转换为 TypeScript：
1. 添加类型注解
2. 定义接口
3. 处理 any 类型

高级功能

条件触发

使用 trigger 字段自动触发技能：

---
name: fix-typo
description: 自动修复拼写错误
trigger:
  - pattern: "typo|spell"
    confidence: 0.8
---

技能组合

在一个技能中引用其他技能：

---
name: full-review
description: 完整代码审查
---

# 完整审查

请依次执行：
1. 使用 code-review 技能
2. 使用 security-review 技能
3. 使用 performance-review 技能

综合以上结果给出最终建议。

变量替换

技能中可以使用变量：

---
name: template
---

# 模板技能

项目名称：{{PROJECT_NAME}}
作者：{{AUTHOR}}
日期：{{DATE}}

技能上下文

技能可以访问以下上下文信息：

变量 说明
{{FILE_PATH}} 当前文件路径
{{PROJECT_DIR}} 项目根目录
{{LANGUAGE}} 文件语言
{{SELECTION}} 选中的文本

技能调试

查看技能信息

/skill list              # 列出所有技能
/skill info <name>       # 查看技能详情
/skill test <name>       # 测试技能

启用调试模式

/config debug true       # 启用调试
/使用 code-review 技能     # 执行时会显示详细信息

技能发布

发布到 npm

# package.json
{
  "name": "claude-skills-myproject",
  "version": "1.0.0",
  "claude": {
    "skills": "./skills"
  }
}

发布到 GitHub

创建仓库并添加以下文件：

claude-skills/
├── skills/
│   └── *.md
├── README.md
└── claude.json

常见问题

Q: 技能文件放在哪里？

A: 放在项目根目录的 .claude/skills/ 目录。

Q: 如何在技能中使用文件内容？

A: 使用 {{FILE_PATH}} 变量或让 Claude 先读取文件。

Q: 技能可以调用命令吗？

A: 可以，在技能中明确告诉 Claude 运行特定命令。

最佳实践

  1. 保持专注 - 每个技能只做一件事
  2. 提供示例 - 包含输入输出示例
  3. 版本管理 - 使用 Git 管理技能变更
  4. 文档完善 - 清晰描述技能用途和用法
  5. 测试验证 - 定期测试技能是否按预期工作
